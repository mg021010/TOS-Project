Step 3

FCFS 스케줄링 및 대기 시간 보정

💡 FCFS 구현 논리

  • 비선점형(Non-preemptive) 실행: 준비 큐(Ready Queue)에 유입된 프로세스가 CPU를 한번 선점하면 실행 종료시까지 양보하지 않고 실행 보장.
  • 정밀한 대기 시간(Waiting Time) 계산 (고득점 포인트):
    • 매 CPU 클록 틱(1초 진행)이 흐르는 시점마다, 현재 상태가 PROCESS_READY인 모든 PCB를 순회하며 대기 시간 필드를 누적 1씩 업데이트하여 측정 오차 방지.
대기시간 계산 공식:
대기 시간(Wait) = 종료 시간(End) - 도착 시간(Arrival) - 총 실행 시간(Total)
create P1 5 create P2 3 set_sched fcfs run stats exit

Step 4

I/O 대기 및 대기 큐(Waiting Queue) 구현

💡 I/O 대기 메커니즘

  • CPU 낭비 방지 (블로킹 모델): 프로세스가 CPU를 점유하는 도중 I/O 요청 발생 시 대기 상태(`PROCESS_WAITING`)로 전이하고 CPU 반환.
  • 대기 큐(Waiting Queue) 흐름:
    • 실행 도중 누적 실행시간(`executed_time`)이 `io_at`에 도달하면 즉시 대기 큐로 이동.
    • 매 틱마다 대기 큐에 속한 프로세스들의 `io_remaining`을 1씩 감소.
    • I/O가 만료(`io_remaining <= 0`)되면 준비 상태(`PROCESS_READY`)로 재활성화하여 Ready Queue의 Tail로 진입.
create P1 5 io 2 3 set_sched fcfs run stats exit

Step 5

Round-Robin (RR) 선점형 스케줄링

💡 시분할 선점 설계

  • 선점(Preemption) 원리: 프로세스의 연속 CPU 사용 시간(`current_quantum`)이 설정된 타임 퀀텀(`rr_quantum`)에 도달 시, CPU 실행 권한 박탈.
  • 상태 천이 및 큐 조정: 해당 프로세스를 `PROCESS_READY`로 환원하고 Ready Queue 맨 뒤로 재삽입, Ready Queue의 Head에 있던 다음 대기 프로세스 디스패치.
  • 문맥 교환 오버헤드 메트릭: 스케줄링 분할 과정에서 발생하는 총 문맥 교환 횟수(`context_switches`)를 카운트하여 효율성 지표 제시.
create P1 5 create P2 3 set_sched rr 2 run stats exit

Step 6

가상 메모리 페이징 및 Page Fault 구현

💡 페이징 및 주소 번역 공식

페이지 번호 = 가상 주소 / 페이지 크기 (128B)
오프셋 = 가상 주소 % 페이지 크기 (128B)
물리 주소 = (물리 프레임 번호 * 128) + 오프셋
  • 물리 메모리 사양: 1KB RAM 크기 (128B 프레임 8개), 가상 페이지 테이블(엔트리 16개).
  • Page Fault 예외 처리: 가상 메모리에 접근(`ACCESS`)하려 할 때, 해당 페이지가 페이지 테이블 상에 매핑(`is_valid=0`)되어 있지 않은 경우 Page Fault 예외 출력.
  • ALLOC & ACCESS 연계 시뮬레이션: `ALLOC`을 통해 가용 프레임을 매핑한 후, 주소 번역이 최종 물리 주소로 성공적으로 완수되는 흐름 검증.
